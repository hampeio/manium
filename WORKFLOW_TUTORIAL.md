# 节点工作流中文教程

## 1. 节点工作流是什么

节点工作流把“素材输入、教学规划、分镜、代码、渲染和输出”拆成可连接的节点。连线代表数据流向；系统按拓扑顺序执行，不允许形成循环。

## 2. 常用节点

- 输入：提示词、图片上传、视频上传、模型选择。
- 参考：风格参考、角色参考、场景参考。
- 规划：图像理解、教学规划、分镜、字幕。
- 控制：镜头运动、时长控制、分辨率设置。
- 逻辑：条件判断、多分支、合并。
- 生成：Manim 代码、渲染、修复。
- 输出：输出节点、预览节点。

节点左侧是输入端口，右侧是输出端口。端口括号中的类型必须兼容，例如 `Prompt → Prompt`、`VideoFile → VideoFile`；`Any` 可以连接任意类型。

## 3. 从零搭建基础工作流

1. 点击节点库中的“提示词”，填写教学主题。
2. 添加“教学规划”和“分镜”节点。
3. 添加“Manim 代码”“渲染”和“输出”节点。
4. 依次连接对应端口。
5. 点击“验证工作流”，先处理缺失输入、端口类型或循环连接错误。
6. 点击“运行工作流”，观察节点状态和阶段进度。

基础示例：

```text
提示词 → 教学规划 → 分镜 → Manim 代码 → 渲染 → 输出
```

带素材与控制的示例：

```text
图片上传 → 图像理解 ┐
提示词 ────────────┼→ 教学规划 → 分镜 → 镜头运动 → Manim 代码
风格参考 ──────────┘                         ↓
时长控制 →─────────────────────────────── 渲染 → 输出
```

## 4. 上传素材

图片、视频节点的参数中保存本地素材路径。素材路径应当存在且可读；移动项目时应同步复制素材。模型密钥不会写入工作流 JSON。

## 5. 节点状态

- 未开始：尚未进入执行队列。
- 等待输入：必填端口没有数据。
- 排队中：上游节点尚未结束。
- 运行中：节点正在处理。
- 已完成：输出已经保存。
- 失败：节点日志会显示具体节点 ID 和错误。
- 已跳过：条件分支未命中。

修改节点参数后，该节点及其下游节点会恢复为“未开始”，下次运行会使用新参数。删除节点时，与它相连的连线会同时删除。

## 6. 条件、分支和合并

条件节点根据 `operator` 与 `expected` 把数据发送到 true 或 false。多分支节点复制输入；合并节点等待上游数据并按 `strategy` 合并。禁止把下游重新连回上游。

## 7. 上传自定义节点

点击“上传自定义节点”，选择 UTF-8 JSON 文件。必填字段：

```json
{
  "type": "MyCustomNode",
  "label": "我的节点",
  "category": "custom",
  "description": "节点用途说明",
  "inputs": [{"name": "input", "type": "Any", "required": false}],
  "outputs": [{"name": "output", "type": "Any", "required": false}],
  "default_params": {"enabled": true}
}
```

`type` 必须以 `Node` 结尾，且不能覆盖内置节点。上传成功后节点会保存到 `custom_nodes`，重启软件后仍会出现。格式错误时界面会指出缺少字段或具体类型错误。

## 8. 调试与常见错误

- “必填输入未连接”：连接对应端口，或为节点提供参数默认值。
- “端口类型不兼容”：检查连线两端的类型。
- “存在循环连接”：删除回指上游的连线。
- “未知节点类型”：重新上传自定义节点配置。
- “渲染超时”：检查场景是否有无限循环、永久 updater 或异常复杂对象。后台会在超时后终止进程并进入修复，不会无限显示“渲染中”。
- “后台任务已中断”：后端曾重启，原任务不会假装继续运行，需要重新提交。

每次执行都会在项目的 `workflow/node_states.json` 和 `workflow_outputs/<节点ID>` 中保存状态、输入快照与输出，便于定位失败节点。
