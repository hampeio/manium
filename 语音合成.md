# 语音合成问题记录

## 本次测试结果

- 输出文件：`C:\Users\面\Desktop\xunfei_tts_test.mp3`
- 测试文本：`我是测试`
- 可用接口：讯飞在线语音合成 WebSocket API
- 请求地址：`wss://tts-api.xfyun.cn/v2/tts`
- 鉴权方式：`APIPassword` 放在请求头 `x-api-key`
- 成功发音人：`xiaoyan`
- 输出格式：`mp3`，请求参数为 `aue=lame` 且 `sfl=1`

## 关键问题

1. `ak-...` 不是旧版 `APIKey/APISecret` 签名里的 `APIKey`，而是新版控制台的 `APIPassword`。
   调用时应放到 WebSocket 请求头：

   ```text
   x-api-key: ak-...
   ```

2. 即使用 `x-api-key` 鉴权，请求体里仍然需要传 `common.app_id`。
   本地 `.env` 里已有 `XUNFEI_APP_ID`，需要和这把 `APIPassword` 配套使用。

3. 不传 `app_id` 或传空 `app_id` 时，服务端可能返回：

   ```text
   10165 invalid handle
   ```

   传错误的 `app_id` 时，会返回类似：

   ```text
   10313 your api_key does not belong to app_id
   ```

4. 本地 `.env` 中旧的 `XUNFEI_API_KEY` / `XUNFEI_API_SECRET` 走 HMAC 签名时返回：

   ```text
   HMAC signature does not match
   ```

   后续不要优先用这组旧签名字段，除非重新确认它们来自同一个讯飞应用且仍有效。

5. `XUNFEI_TTS_VOICE=x4_xiaoyan` 在本次测试中只返回了极短音频片段。
   建议先用 `xiaoyan` 做基础测试；需要换音色时，先确认该 `app_id` 已授权对应发音人。

6. PowerShell 通过 here-string 管道传 Python 脚本时，中文源码可能被转成 `????`。
   为避免测试文本被错误编码，脚本里建议使用 Unicode 转义：

   ```python
   text = "\u6211\u662f\u6d4b\u8bd5"
   ```

   或者把脚本保存为 UTF-8 文件后再执行。

## 推荐最小请求结构

```json
{
  "common": {
    "app_id": "从 .env 读取 XUNFEI_APP_ID"
  },
  "business": {
    "aue": "lame",
    "sfl": 1,
    "auf": "audio/L16;rate=16000",
    "vcn": "xiaoyan",
    "speed": 50,
    "volume": 50,
    "pitch": 50,
    "tte": "UTF8"
  },
  "data": {
    "status": 2,
    "text": "base64(UTF-8 文本)"
  }
}
```

## 排查顺序

1. 先检查 `x-api-key` 是否使用 `APIPassword`，不要误用旧版 `APIKey`。
2. 确认请求体包含正确的 `common.app_id`。
3. 先用 `xiaoyan` 测通，再换其他发音人。
4. 检查待合成文本是否真的按 UTF-8 编码传入，不要变成 `????`。
5. 生成后看文件大小；四个中文字正常不应只有几百字节。
6. 如需确认文件有效，可检查文件头是否为 MP3 帧：`ff f3` / `ff fb` 或 `ID3`。
